# Подключение камеры (Hikvision + ваша моделька)

Hikvision даёт **видео**, ваша моделька **считает мешки / читает номер**, и в
конце шлёт результат на наш **вебхук**. Моделька и сервер — на одном ПК.

```
Hikvision (RTSP видео) → ваша моделька → POST /api/webhook/camera/ → ответ сервера
```

## Шаг 1. Узнать RTSP-адрес камеры

С логином/паролем Hikvision поток открывается так:
```
rtsp://<логин>:<пароль>@<IP_камеры>:554/Streaming/Channels/101
```
- `101` — основной поток первого канала (`102` — субпоток, легче для CV).
- `<IP_камеры>` — локальный IP камеры в сети цеха (не публичный 2.132.77.239 —
  это облачный адрес Hik-Connect).

## Шаг 2. Создать камеру в системе и получить ключ

Фронт → **Управление → Камеры → Добавить камеру** (тип «Счётчик загрузки»,
ID `counter-01`). Скопируйте показанный ключ.

Либо используйте авто-регистрацию: моделька шлёт enrollment-ключ
(`CAMERA_ENROLL_KEY` из `.env`) с новым `camera_id` → камера появится в
«Обнаруженных», привяжите её в UI и получите личный ключ.

## Шаг 3. Вставить вызов в модельку

Скопируйте `camera_client.py`, впишите `CAMERA_ID` и `CAMERA_KEY`, и в конце
обработки машины вызовите:
```python
result = send_to_server(plate="777ABC02", bags=50)
if result.get("allowed"):
    open_gate()   # решение приходит от сервера
```

## Шаг 4. Проверка без камеры

```bash
pip install requests
python integrations/camera_client.py --plate 777ABC02 --bags 50
```
Сервер ответит по шаблону камеры, напр. `{"decision": "allow", ...}`.

## Что отправлять по типу камеры

| Тип камеры   | Поля запроса                         | Что делает сервер           |
|--------------|--------------------------------------|-----------------------------|
| Въезд        | `plate`                              | проверка заказа → `arrived` |
| Счётчик      | `plate`, `bags`                      | `loading` + мешки           |
| Выезд        | `plate`, `weight_kg`                 | `shipped` + нетто/списание  |

## Счётчик мешков (живой счёт)

CV-воркер шлёт `+1` на каждый посчитанный мешок:
```python
for event in counter.run(rtsp_url):
    requests.post("http://localhost:8000/api/webhook/camera/",
                  json={"camera_id": "counter-01", "increment": 1},
                  headers={"X-Camera-Key": "<ключ>"})
```
Счёт виден в CRM → **Управление → Счётчик мешков**. Оператор вводит номер
машины и жмёт «Закончить сессию» — итог уходит в заказ (статус `loading`).
