# Живой AI-подсчёт мешков на посту погрузки

**Дата:** 2026-07-09
**Статус:** утверждён к реализации

## Цель

На шаге «Погрузка» контролёр одним касанием включает AI-модель на камере зоны
загрузки: плеер переключается на аннотированный поток (боксы, линия, счётчик
поверх видео), рядом с ручным счётчиком появляется живое AI-число с весом и
раскладкой по цветам. Кнопка «Принять» переносит AI-число в ручной счётчик.
«Погрузка завершена» выключает модель — GPU освобождается.

Ручной счётчик остаётся источником истины (`Shipment.bags_loaded`); AI — сверка
и ускорение, как задумано в сценарии ai_service.

## Контекст

AI-сервис (`ai_service.py`) развёрнут на ПК с камерами (Tailscale
`100.109.156.107:8890`, ключ `X-Api-Key`) и включается per-камера по HTTP.
Аннотированный поток публикуется в MediaMTX как `cam<N>ai` (уже H.264).

Планшет контролёра **не в Tailscale** — ни API AI-сервиса, ни MediaMTX ему
недоступны напрямую. Оба канала обязаны идти через сервер:

```
Планшет ── https://asyl-ltd.kz ──► nginx ──► Django /api/cameras/<n>/ai/ ──► ai_service :8890
                                      └────► go2rtc (cookie-auth) ──rtsp──► MediaMTX camNai
```

Это же решает секрет: ключ API живёт только в окружении бэкенда.

## Бэкенд (`apps/cameras`)

### `ai.py` — клиент AI-сервиса (новый модуль)

- `CAMERA_AI_URL` (дефолт `http://{CAMERA_HOST}:8890`), `CAMERA_AI_KEY` (без
  дефолта; пусто → фича выключена, эндпоинты отвечают 503 `ai_disabled`).
- stdlib `urllib` (requests в зависимостях нет), таймаут 10 с, заголовок
  `X-Api-Key`.
- Функции: `status(n) -> dict | None` (None при 404 — не запущена),
  `start(n) -> dict`, `stop(n) -> dict | None` (сначала GET финального статуса,
  потом DELETE — счётчик живёт в памяти обработчика), `reset(n) -> dict`.
- Ошибки: `AiUnavailable` (сеть/таймаут → 502), `AiError(status, detail)`
  (401 ключ / 409 лимит — пробрасываем осмысленно).

### Вьюхи и маршруты

| Метод | Путь | Право | Действие |
|---|---|---|---|
| GET | `/api/cameras/<n>/ai/` | staff | статус+счётчик; не запущена → `{"running": false}` |
| POST | `/api/cameras/<n>/ai/` | `shipping.load` | включить; уже работает → вернуть статус как есть (не сбрасывая счёт) |
| DELETE | `/api/cameras/<n>/ai/` | `shipping.load` | финальный статус + выключение |
| POST | `/api/cameras/<n>/ai/reset/` | `shipping.load` | обнулить счётчик (новая погрузка) |

POST идемпотентен к повторному включению (второй планшет, перезагрузка
страницы) — присоединяемся к работающему обработчику, счёт не трогаем.

## go2rtc

`camNai` — уже H.264, транскод не нужен: нативный RTSP-рестрим
(`camNai: rtsp://viewer:…@CAMERA_HOST:8554/camNai`), пре-провижен cam1..cam32
по образцу основных потоков. Поднимается по требованию — простаивающие потоки
ничего не стоят. Пока модель греется (5–15 с), MediaMTX отдаёт 404 — go2rtc и
MSE-плеер переподключаются сами (существующий бэкофф в `CameraStream`).

## Фронтенд (пост погрузки)

### Хук `use-ai-counter.ts`

`useAiCounter(camId, enabled)` — статус AI-камеры: GET при входе на шаг,
поллинг раз в 1.5 с пока запущена (и вкладка видима), действия
`start/stop/reset` с `busy`/`error`. Тип `AiStatus` — как отдаёт ai_service
(`total`, `weight`, `per_color`, `fps`, `status`, `stream`).

### Привязка к камере

AI привязывается к камере зоны загрузки (`zone` содержит «загруз»; fallback —
показанная камера), а не к той, что контролёр листает в данный момент: переключение
вида на «Ворота» не прячет счётчик. Плеер показывает `${cam.src}ai`, только
когда выбранная камера совпадает с AI-камерой и модель запущена.

### UX шага «Погрузка»

- Под ручным счётчиком — панель AI:
  - выключена → кнопка «AI-подсчёт по камере»;
  - включается → «запуск модели…» (первый прогрев до минуты);
  - работает → крупное AI-число, вес (кг), чипы `per_color`, кнопки
    «Принять N» (пишет в существующий `/orders/<id>/load/`) и «Выкл».
- Значок «AI» с живой точкой поверх видео, когда плеер на AI-потоке.
- «Погрузка завершена» → после `finish-loading` выключаем AI (ошибку стопа
  глотаем — переход важнее), плеер возвращается на обычный поток.
- Ручной счётчик подтянет принятое число через существующую логику
  «чужих обновлений» (поллинг `order.bags_loaded`).

## Что не делаем (v1)

- Автостарт AI при приёме машины — жжёт GPU в простое, лимит 4 камеры;
  включение остаётся за контролёром (одно касание).
- Автозапись AI-числа в `bags_loaded` — только явное «Принять».
- Хранение AI-сессий в БД — состояние живёт в ai_service и опрашивается.
- Настройка линии/направления per-камера — дефолты ai_service; клиент `start`
  принимает словарь опций на будущее.

## Тесты

`apps/cameras/tests/test_ai.py`: мок `ai._request`/клиента — статус «не
запущена», идемпотентный старт, stop возвращает финал до DELETE, 503 без ключа,
502 при недоступном сервисе, 409 лимита пробрасывается, права (staff / 
`shipping.load`).

## Проверка

Локальный мок ai_service (маленький HTTP-скрипт с инкрементом счётчика),
`CAMERA_AI_URL` бэкенда на него — полный клик-тест поста погрузки в превью:
включение, живое число, «Принять», завершение погрузки со стопом.

## Дополнение (2026-07-09, вечер): список камер из инвентаря `GET /cameras`

У ai_service появился живой инвентарь сети (скан каждые 5 минут): каналы
NVR, direct-камеры (стабильный путь `cam_<hex>` по MAC) и «locked»
(в сети, но пароль неизвестен). Сайт строит список из него, а не из
перебора 32 слотов:

- `discover_cameras()` — сначала `ai.inventory()`; резерв прежний
  (RTSP-пробы cam1..cam32), если ai_service недоступен. Кэш тот же.
- Payload камеры: `id` — стабильный `kind:mac` (не номер канала!),
  `src: string | null` (имя потока go2rtc; null у locked), `kind`,
  `online` из инвентаря, `note` у locked.
- AI-прокси адресует камеры путями: `/api/cameras/cam2/ai/`,
  `/api/cameras/cam_8c26/ai/` (валидация `^cam\w{1,16}$`, цифры
  нормализуются в `camN`).
- go2rtc: статик-слоты cam1..cam32 остаются; для путей вне них (direct,
  camN>32) бэкенд при discovery дозаявляет потоки через внутренний API
  (`GO2RTC_API_URL`, в проде `http://go2rtc:1984`): `{path}src` →
  ffmpeg-транскод `{path}` → и `{path}ai`. go2rtc держит их до рестарта,
  следующий discovery заявит заново; ошибка sync не валит список.
- Фронт: `CameraFeed.id: string`, locked-плитки «Нет доступа» (замок, IP,
  note в title) на стене камер, счётчик «N из M онлайн · K без доступа»;
  пост погрузки работает только с играбельными камерами, AI-хук принимает
  путь камеры. «Принять» заблокирована, пока модель на прогреве.
