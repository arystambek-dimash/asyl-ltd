# Рестайл форм в модалках под qoima

**Дата:** 2026-06-23
**Статус:** утверждён к реализации
**База:** main

## Цель

Привести формы в модалках к стилю qoima: шапка с eyebrow + крупным заголовком +
описанием и разделителем, секции с подписями, аккуратные поля (Label обычного
регистра + hint), футер с верхним разделителем и кнопками справа. Прокачать
переиспользуемые компоненты (Modal/Field/Label), затем перевести все 8 форм.

## Компоненты

### `Modal` (`ui/modal.tsx`) — расширение пропсов
Добавить опциональные пропсы (всё backward-compatible — старые `<Modal title>` работают):
- `eyebrow?: string` — мелкая строка над заголовком (12px, muted).
- `description?: string` — подзаголовок под заголовком (14px, muted).
- `footer?: React.ReactNode` — если задан, рендерится отдельным блоком внизу с
  верхним разделителем и лёгким фоном (`bg-[var(--muted)]/40`), контент
  выравнивается вправо. Тело формы при этом без своего футера.
Шапка: `eyebrow` → крупный заголовок (`text-[22px] font-bold tracking-tight`) →
`description`, под ними `border-b`. Тело — прокручиваемое (`overflow-y-auto`),
футер — фиксированный блок (вне прокрутки), как у qoima.

### `Field` (`ui/field.tsx`) — новый
```
{ label?: ReactNode; hint?: ReactNode; children: ReactNode; className?: string }
```
Рендерит `Label` (если есть) сверху, `children`, и `hint` (12px muted) снизу.
Заменяет повторяющийся `<div className="grid gap-2"><Label>…</Label><Input/></div>`.

### `Label` (`ui/label.tsx`) — обычный регистр
Сейчас: `text-xs font-semibold uppercase tracking-[0.08em] muted`. Меняем на
qoima-стиль: `block text-[12px] font-medium text-[var(--foreground)] mb-1.5`
(обычный регистр, не uppercase). Input/Select не трогаем (фокус-кольцо уже синее
после рестайла палитры).

## Формы (перевод на новый Modal + Field)

Все 8 форм: вынести кнопки в `footer`, использовать `Field` для полей, добавить
`eyebrow`/`description`, секции где уместно. Логика (стейт, submit, валидация,
права) не меняется.

| Форма | eyebrow | секции |
|---|---|---|
| Сотрудники (новый) | «Команда · Сотрудник» | Человек / Должность / Роль |
| Клиенты (новый) | «Работа · Клиент» | Контакты / Реквизиты |
| Заказы (новый) | «Работа · Заказ» | Клиент / Позиции |
| Товары (новый) | «Номенклатура · Товар» | — (4 поля) |
| Роли (создать/изм.) | «Доступы · Роль» | Название / Права |
| Камеры (создать) | «Управление · Камера» | — |
| Камеры (привязать) | «Управление · Привязка» | — |
| Склад (изменить остаток) | «Склад · Корректировка» | — |

Секция = `<section className="space-y-3 pt-4 border-t border-[var(--border)]">`
(первая секция без `pt-4 border-t`) с подписью
`<h4 className="text-[12px] font-medium text-[var(--muted-foreground)]">…</h4>`.

Футер (через проп `footer`): `<><Button variant="outline">Отмена</Button>
<Button type="submit" form="…">…</Button></>` — выравнивание вправо делает Modal.
(Поскольку submit-кнопка уезжает из `<form>` в футер Modal, формам дать `id` и
кнопке `form={id}`, либо вызывать submit через onClick — выбрать рабочий вариант
на форму; для простоты использовать `form={id}` на submit-кнопке.)
