# Редизайн детальной страницы заказа с чартами

**Дата:** 2026-06-23
**Статус:** утверждён к реализации
**База:** main

## Цель

Детальная заказа сейчас выглядит пусто и непонятно (узкие карточки справа, левая
половина пустая). Переделать в сбалансированный 2-колоночный дашборд с чартами
(recharts уже в проекте), степпером статуса и нормальными лейблами товаров.
Логика не меняется — только разметка/визуализация.

## Структура

**Шапка:** «Заказ #N» + клиент/номер машины + `StatusBadge`, и **степпер статуса**
по полному жизненному циклу: черновик → подтверждён → оплачен → прибыл → загрузка
→ загружен → отгружен. Текущий — подсвечен, пройденные — зелёные. `cancelled` —
отдельная пометка (бейдж «Отменён», степпер не рисуем или серый).

**Левая колонка (шире, lg:col-span-2):**
- **Позиции**: товар (`product_label`, не «Товар #N») × мешков + «Сумма заказа».
- **Оплата**: кольцевой чарт (RadialBar) оплачено/остаток, в центре % оплаты;
  под ним суммы (оплачено/остаток) + поле «внести оплату» (если `payments.create`).
- **Мешки** (только при `shipped`): столбчатый чарт (BarChart) Заказано / Камера /
  По весу — наглядное сравнение трёх чисел.

**Правая колонка (уже):**
- **Веса** (при наличии shipment): въезд / выезд / груз — строки + мини-полоски.
- **Действия**: подтвердить заказ / статус-сообщения — как есть.
- **Сменить статус** (если `orders.edit`) — как есть.

## Тех-детали
- Чарты: recharts, `ResponsiveContainer`, фикс-высота (~180px кольцо, ~200px бары),
  цвета через CSS-переменные (`var(--ring)` синий = оплачено/факт, `var(--success)`
  зелёный, `var(--muted)` остаток/фон).
- `product_label` уже в `OrderItemSerializer` — использовать его в позициях.
- Адаптив: `grid lg:grid-cols-3`; на узком — одна колонка.
- Формула «отгружено по весу» = `net ÷ bag_weight_kg` (как в текущей карточке итога).
- Степпер: переиспользовать/повторить компонент (как `Stepper` в shipping/page.tsx),
  но с полным набором статусов заказа.

## Тестирование
- `npm run build` — без ошибок.
- Docker `up --build`: открыть заказы в статусах черновик / оплачен / отгружен —
  степпер, кольцо оплаты, бары мешков (shipped), веса, действия; light + dark;
  узкий экран (колонки в одну). Бэкенд не трогаем.

## Вне scope (YAGNI)
- Новые эндпоинты/поля БД (всё уже отдаётся: weights, bags_loaded, bag_weight_kg,
  paid_total, total_amount).
- Изменения поста отгрузки / списка заказов.
- Чарты вне детальной заказа.
- Изменение логики оплаты/подтверждения/смены статуса.
