# Раскатка стиля qoima на основные списки

**Дата:** 2026-06-23
**Статус:** утверждён к реализации

## Цель

Применить уже готовые компоненты (StatCard, FilterPills, SortableHeader) и
паттерн страницы «Заказы» на основные внутренние списки: Клиенты, Склад, Каталог
(сорта/упаковки/продукты), Сотрудники. Каждая страница получает верхний ряд
stat-карточек, поиск/фильтр (где уместно) и сортируемые заголовки. Вся
существующая логика (toggle активности, adjust склада, inline-edit цены,
создание в модалке, проверки прав) сохраняется без изменений.

## Принцип

Единый паттерн (как на Заказах): `AppShell` → ряд `StatCard` (grid sm:grid-cols-N)
→ строка с поиском + (опц.) `FilterPills`/`Button` → `Card`+`Table` с
`SortableHeader`. Компоненты уже существуют (`ui/stat-card`, `ui/filter-pills`,
`ui/sortable-header`) — только применяем. Бэкенд не трогаем. Reports/Events/
Камеры/Роли/Portal — вне scope (свой формат).

## Страницы

### Клиенты (`app/clients/page.tsx`)
- StatCard: «Всего клиентов».
- Поиск по имени/телефону/стране (клиентская фильтрация).
- Сортировка: Имя, Телефон. Колонки те же (Имя/Телефон/Страна).
- Кнопка «Добавить клиента» + модалка `ClientForm` — без изменений.

### Склад (`app/warehouse/page.tsx`)
- StatCard ряд (3): «Позиций», «Мешков», «Тонн» (accent). Заменяют текущую
  текстовую сводку (totalBags/totalTons считаются как сейчас, из `filtered`).
- Существующие поиск + селекты сорт/фасовка — оставить как есть (они уже есть).
- Сортировка по: Товар (product_label), Остаток (bags). Остальные колонки/Badge
  статуса (`stockTone`) и модалка adjust — без изменений.

### Каталог / Сорта (`app/catalog/grades/page.tsx`)
- StatCard (2): «Всего», «Активных» (is_active).
- Сортировка: Сорт (name). Toggle-кнопка и модалка создания — без изменений.

### Каталог / Упаковки (`app/catalog/packagings/page.tsx`)
- StatCard (2): «Всего», «Активных».
- Сортировка: Фасовка (name), Вес (weight_kg). Toggle + модалка — без изменений.

### Каталог / Продукты (`app/catalog/products/page.tsx`)
- StatCard (2): «Всего», «Активных».
- Сортировка: Товар (label), Цена (price). Inline-edit цены, toggle, «ready»-
  проверка, модалка — без изменений.

### Сотрудники (`app/management/employees/page.tsx`)
- StatCard (2): «Всего», «Активных».
- Поиск по имени/логину/должности. Сортировка: Имя (name), Роль (role_name).
  Колонки и модалка создания — без изменений. Проверка `canManage` сохраняется.

## Реализация (общий рецепт на страницу)

1. Импортировать `StatCard`, при нужде `FilterPills`/`Search`, `SortableHeader` +
   `type SortDir`.
2. Добавить состояния `q` (поиск, где вводим), `sortKey`, `sortDir` + `toggleSort`.
3. Считать список из хука; добавить `filtered` (поиск/фильтр) и `sorted`
   (по ключу/направлению), как на Заказах. Существующие фильтры склада не дублировать.
4. Верхний ряд `StatCard` вместо/рядом текстового счётчика.
5. В `THead` заменить нужные `TH` на `SortableHeader`; рендерить `sorted` вместо
   сырого списка.
6. Никакую существующую логику (toggle/adjust/inline-edit/модалки/права) не менять.

## Тестирование

- `npm run build` — без ошибок и типовых ошибок после каждой страницы.
- Визуальная проверка в Docker (`docker compose up --build`): открыть Клиенты,
  Склад, по одной странице каталога, Сотрудники — убедиться: stat-карточки сверху,
  сортировка работает (стрелки), поиск фильтрует, существующие действия
  (toggle/adjust/inline-цена) работают. Light + dark.
- Юнит-тестов на верстку нет.

## Вне scope (YAGNI)

- Детальные страницы `[id]` и редактирование строк (отдельная фича — модалки).
- Portal, Reports, Events, Камеры, Роли.
- Новые колонки/поля, прогресс-бары (нет процентных данных на этих сущностях).
- Изменения бэкенда.
