# Кнопки создания в топбаре (как qoima)

**Дата:** 2026-06-23
**Статус:** утверждён к реализации

## Цель

Перенести кнопки создания/действия из тела каждой страницы в **топбар**
(контекстно для текущей страницы), как сделано в qoima. Кнопка остаётся
страничной (её `onClick`/модалка/право — на странице), но визуально
располагается в топбаре справа.

## Подход

qoima не имеет глобального меню «+» — у них кнопка создания живёт в `Topbar`
через проп `actions`. Повторяем: добавляем слот `actions` в наш `Topbar` и
`AppShell`, каждая страница передаёт свою кнопку через `<AppShell actions={...}>`.
Логика создания (локальный `open`-стейт, модалка, `can(...)`-гейт) не меняется —
двигается только сам элемент кнопки.

## Компоненты

### `Topbar` (`components/layout/topbar.tsx`)
- Добавить проп `actions?: React.ReactNode`.
- Рендерить `actions` в правом блоке, перед `ThemeToggle` (с отступом).

### `AppShell` (`components/layout/app-shell.tsx`)
- Добавить проп `actions?: React.ReactNode`.
- Пробросить в `<Topbar ... actions={actions} />`.

## Страницы (перенос кнопки из тела в `AppShell actions`)

Для каждой: вынести существующую кнопку в `actions`, убрать её из тела
(оставив поиск/фильтры/stat-карточки на месте). Стейт модалки и gate — без изменений.

| Страница | Кнопка | Право |
|---|---|---|
| Заказы | «Новый заказ» | `orders.create` (canCreate) |
| Клиенты | «Добавить клиента» | без гейта |
| Каталог/Сорта | «Добавить сорт» | без гейта |
| Каталог/Упаковки | «Добавить фасовку» | без гейта |
| Каталог/Продукты | «Создать товар» (disabled !ready) | без гейта |
| Сотрудники | «Добавить сотрудника» | `employees.manage` (canManage) |
| Роли | «Новая роль» | без гейта |
| Камеры | «Добавить камеру» | `cameras.manage` |
| Склад | «Изменить остаток» | `warehouse.adjust` (canAdjust) |

После переноса: если в теле страницы кнопка стояла в одной строке с поиском/
счётчиком — строку оставляем (поиск/фильтры), только убираем кнопку. Если после
удаления кнопки строка становится пустой — убрать пустой контейнер.

Portal: `portal/orders` кнопка «Новый заказ» (ссылка на `/portal/orders/new`) —
тоже перенести в топбар (portal использует тот же AppShell).

## Тестирование

- `npm run build` — без ошибок после каждой страницы.
- Визуальная проверка в Docker: на каждой странице кнопка создания видна в
  топбаре справа (перед переключателем темы), модалка/переход работает, право
  скрывает кнопку где нужно. Light + dark, и мобильная ширина (кнопка не ломает
  топбар — текст можно прятать на узком экране через `hidden sm:inline`).

## Вне scope (YAGNI)

- Глобальное меню «+» с выпадающим списком сущностей.
- Вынос форм в общие компоненты.
- Командная палитра / поиск ⌘K.
- Изменения бэкенда.
